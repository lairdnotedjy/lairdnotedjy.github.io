npm init 新建(/更新) package.json

如果package.json文件依据存在不会改变未设置的配置项

奇淫技巧

方式一: 删除package.json中的dependencies项,再npm init会依据node_modules的实际安装情况,更新dependencies配置项.但会多余配置部分信息,可能与原文件不一致,会出现覆盖.

方式二: 为减少误操作,可先备份(或重命名)package.json,再 npm init同样会 依据node_modules的实际安装情况,更新dependencies配置项

如何把node_module所有模块的版本信息更新到package.json中？
重新执行npm init
